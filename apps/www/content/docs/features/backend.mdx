# Backend Support

Monoframe generates Express + TypeScript backends that integrate seamlessly with the monorepo.

## What You Get

Each backend app includes:

- **Express** web server
- **TypeScript** with shared config from `@repo/typescript-config`
- **tsx** for hot-reload during development
- Proper `build` and `dev` scripts
- Port configuration to avoid conflicts with frontend apps

## Project Structure

```
apps/api/
├── src/
│   ├── index.ts          # Express server entry point
│   └── routes/
│       └── health.ts     # Generated health route
├── package.json
└── tsconfig.json         # Extends @repo/typescript-config
```

## Development

```bash
pnpm dev  # Starts all apps including backends
```

Backend apps run on port `4000` by default (incrementing for additional backends).

## Entry Point

The generated `src/index.ts`:

```typescript
import express from 'express';
import cors from 'cors';
import healthRouter from './routes/health.js';

const app = express();
const PORT = process.env.PORT || 4000;

app.use(cors());
app.use(express.json());

app.use('/health', healthRouter);

app.listen(PORT, () => {
  console.log(`[api] Server running on http://localhost:${PORT}`);
});
```

## Multiple Backends

You can create multiple backend apps during setup. Each gets its own port, directory, and configuration — all sharing the same TypeScript and code-quality configs.
